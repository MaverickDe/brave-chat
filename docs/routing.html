forward Routing

This project uses react-router v6 as its base routing system. You can find all project's routes configuration in the src/routes folder. If you want to define a new route to the project, you can follow the steps below:

  1. Import your react component into src/routes/index.js file.
    2. import newComponent from "../pages/newPage"
  2. Use the Route component to define your new route.
    3. <Route ...props> </Route>
  3. Specify the path and component props for the Route component.
    4. <Route exact path="/new-url" > </Route>
  4. Add the new Route component to the project's root Routes component.
    5. <Routes> <Route exact path="/new-url" element={ <newComponent />} > </Route> </Routes>

  5. Or you can lazy load your component; Lazy loading is a great way to improve the performance of your React application. By loading components on-demand, you can avoid wasting time and resources on components that the user may never even see.

    To lazy load a React component, you can use the dynamic import() syntax. This will tell React to only load the component when it is needed.

    import { lazy } from 'react'; <Routes> <Route exact path="/new-url" element={lazy(() => import('../pages/newPage'))} > </Route> </Routes>

    This can greatly improve the performance of your application!

  6. That's it! Your new route should now be accessible from the project URL: localhost:3000/new-url .

Once you have git, Node.js and one of the package managers installed, you should be good to go! Now you need to fork and clone the repository:
  • Fork the Chat repo on GitHub.
  • Clone your fork locally: git clone git@github.com:your_name_here/chat.git
  • Open the newly created directory: cd chat
Now, in order to run the project locally or build for production use, you will need to run the following commands:
Command Description
npm install or yarn install This command will install everything that's needed in the node_modules folder, so you won't have to worry about doing it manually. Plus, it's a lot faster than installing each dependency individually.
npm start or yarn start This command will start the server at http://localhost:3000 and watch for any changes in your code. This is a great way to develop and test your code before putting it live.
npm build or yarn build This command will generate a /dist directory containing all of the files you need to deploy your project.
Each of these commands serves a specific purpose and is essential for getting the project up and running smoothly. So be sure to run the commands in the order listed above.
